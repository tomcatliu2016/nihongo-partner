# 製品要件定義書（PRD）

## NihongoPartner

**Agentic AI 日本語学習アシスタント**

---

## 1. プロジェクト概要

### 1.1 プロジェクト紹介

NihongoPartner は、Google Cloud AI を活用した自律型日本語学習エージェントです。実践的なシーン会話練習、スマート分析、個別推薦を通じて、外国人学習者が効率的に日本語会話能力を向上できるよう支援します。

### 1.2 コアバリュープロポジション

| 価値 | 説明 |
|------|------|
| **自律的意思決定** | エージェントがユーザーのパフォーマンスに基づいて自律的に教育戦略を調整、人的介入不要 |
| **リアルタイムインタラクション** | 音声会話練習で実際のシーンをシミュレート |
| **個別学習** | ユーザーの弱点に基づいてカスタマイズされた学習教材と練習パスを生成 |

### 1.3 Hackathon 目標

- 多言語理解と自律的意思決定における Gemini の能力を実証
- エージェントの「自律性」（Agentic 特性）を体現
- Google Cloud 技術要件を満たし、オンライン体験可能な MVP を完成

> 📝 **注記**：今回の Hackathon はオンライン提出形式で、現場プレゼンテーションは不要です。提出物にはコードリポジトリ、製品ドキュメント、デモ動画が含まれます。

### 1.4 長期ビジョン

NihongoPartner は Hackathon 出品作品にとどまらず、長期的に発展する製品です。Hackathon 後も継続的にイテレーションします：

- **短期（1-2 ヶ月）**：コア機能の完善、ユーザー体験の最適化
- **中期（3-6 ヶ月）**：シーンライブラリの拡張、ソーシャル機能の追加
- **長期（6 ヶ月以上）**：商業化運営、他言語への展開

### 1.5 多言語サポート

アプリケーションインターフェースは 3 言語をサポートし、ユーザーは自由に切り替えできます：

| 言語 | コード | 説明 |
|------|--------|------|
| 簡体字中国語 | zh | デフォルト言語 |
| 日本語 | ja | 上級学習者向け |
| 英語 | en | 国際ユーザー向け |

## 2. 市場分析

### 2.1 ターゲット市場

**主要市場**：
- 中国から日本への留学生および就労者
- グローバル日本語学習者（アジア太平洋地域中心）

**市場規模**：
- グローバル日本語学習者は約 380 万人（2023 年 JLPT 受験者データ）
- 中国は最大の日本語学習市場、約 30% を占める

### 2.2 競合分析

| 競合 | 強み | 弱み | 当社の差別化 |
|------|------|------|--------------|
| **Duolingo** | ゲーミフィケーション設計、大規模ユーザーベース | 実際の会話練習の欠如、音声インタラクションなし | 実践的シーン会話、音声インタラクション |
| **HelloTalk** | ネイティブパートナー、ソーシャル性 | 相手の時間に依存、品質が不安定 | 24/7 利用可能、AI の一貫性 |
| **JapanesePod101** | 豊富なコンテンツ、専門性が高い | 受動的学習、インタラクション不足 | 能動的練習、即時フィードバック |
| **Rosetta Stone** | 没入型学習 | 高価格、個別化の欠如 | AI 個別化、オンデマンドコンテンツ生成 |
| **italki** | ネイティブ講師、柔軟性 | 高コスト、予約が必要 | 無料/低価格、いつでも利用可能 |

**当社のコア差別化**：
1. **Agentic AI** - 自律的意思決定を行う AI コーチ、単純な Q&A ではない
2. **リアルタイム音声会話** - 実際のシーンをシミュレート、「話せない日本語」を克服
3. **スマート診断** - 弱点を自動識別、ターゲットを絞った向上
4. **オンデマンド生成** - 学習教材をリアルタイム生成、真の個別化

## 4. 機能要件

### 4.1 コア機能（MVP - Must Have）

#### 機能 1：シーン会話練習 🎯 **最高優先度**

**機能説明**

ユーザーが生活シーンを選択し、AI エージェントと実際の会話練習を行います。エージェントはシーン内の役割（ウェイター、店員など）を演じ、ユーザーは日本語でインタラクションします。

**エージェント自律行動**
- ユーザーの日本語レベルをリアルタイム評価
- パフォーマンスに応じて会話難易度を自律調整
- ヒントや助けを提供するタイミングを判断
- 現在のシーンを終了するタイミングを決定

**対応シーン（MVP）**

| シーン | 役割 | 典型的な会話内容 | 難易度範囲 |
|--------|------|------------------|------------|
| レストラン注文 | ウェイター | 注文、おすすめ確認、会計 | 1-5 |
| ショッピング | 店員 | 価格確認、試着、支払い | 1-5 |
| 自己紹介 | 面接官/新しい同僚 | 基本情報、趣味、職歴 | 1-5 |

**ユーザーストーリー**
```
日本語学習者として
レストラン注文シーンで会話練習をしたい
実際の状況でスムーズに対応できるようになるために
```

**受け入れ基準**
- [ ] 3 つの基本シーン（レストラン、ショッピング、自己紹介）をサポート
- [ ] 音声入力とテキスト入力をサポート
- [ ] エージェント音声応答（速度調整可能）
- [ ] 会話中にエージェントがヒントを出すか自律的に判断
- [ ] エージェントがユーザーのエラーを認識しつつ会話の流れを中断しない
- [ ] 1 回の会話で 5-8 ターンのインタラクション
- [ ] 会話応答時間 < 3 秒

**インタラクションフロー**
```
シーン選択 -> 難易度選択 -> 会話開始
                              ↓
              ┌───────────────────────────────┐
              │  ユーザー入力（音声/テキスト）    │
              │         ↓                      │
              │  エージェント分析（文法、語彙、文脈） │
              │         ↓                      │
              │  エージェント自律判断            │
              │  - 会話を継続                  │
              │  - 難易度を下げる + ヒント       │
              │  - 難易度を上げる              │
              │  - エラーを修正                │
              │  - 会話を終了                  │
              │         ↓                      │
              │  エージェント応答（音声 + テキスト） │
              └───────────────────────────────┘
                              ↓
                        会話終了 -> 分析レポート生成
```

#### 機能 2：スマート学習分析 🎯 **高優先度**

**機能説明**

会話終了後、エージェントが自動的にユーザーのパフォーマンスを分析し、弱点を特定して学習レポートを生成します。

**エージェント自律行動**
- エラータイプの統計と分類
- 文法の弱点を特定
- 全体的な進歩傾向を評価
- 改善提案を自律生成

**ユーザーストーリー**
```
日本語学習者として
自分がどこができていないかを知りたい
的確に改善するために
```

**受け入れ基準**
- [ ] 会話中の主なエラー（3-5 個）を表示
- [ ] エラータイプの分類（文法/語彙/語順など）
- [ ] 具体的な改善提案を提供
- [ ] 良かった点を表示（ポジティブな強化）
- [ ] 視覚化されたスコアを生成

**分析レポート構造**
```
📊 会話分析レポート

総合スコア：78/100  ⭐⭐⭐⭐

✅ 良かった点：
• 丁寧語の使用が正確（です/ます形）
• 基礎語彙の習得が確実

⚠️ 改善が必要：
• 助詞「に」「で」の混同（エラー 3 回）
  - ✗ レストランに食べます
  - ✓ レストランで食べます
• 動詞て形の活用が不慣れ（エラー 2 回）

💡 エージェントからの提案：
助詞の使い方を重点的に復習してください。専用練習コンテンツを用意しました。
明日は「ショッピング」シーンで練習し、助詞の使用を継続して強化することをお勧めします。

[学習教材を見る] [もう一度練習] [別のシーンへ]
```

#### 機能 3：個別学習教材 🎯 **高優先度**

**機能説明**

ユーザーの弱点に基づいて、エージェントが的確な学習教材をリアルタイムで生成します。

**エージェント自律行動**
- 弱点に基づいて解説コンテンツをリアルタイム生成
- 的確な練習問題を作成
- 外部学習リソースを推薦
- ユーザーレベルに合わせた難易度

**ユーザーストーリー**
```
日本語学習者として
自分の弱点に対応した学習教材を得たい
迅速に向上するために
```

**受け入れ基準**
- [ ] 簡潔な文法解説を生成（ユーザーインターフェース言語で）
- [ ] 例文を提供（3-5 個、易から難へ）
- [ ] 的確な練習問題を生成（3-5 問）
- [ ] 外部リソースを推薦（2-3 リンク）
- [ ] 教材生成時間 < 5 秒

**生成コンテンツ構造**
```
# 助詞「に」と「で」の違い

## コア解説
「に」は目的地や時点を表します
「で」は動作が発生する場所や方法を表します

## 例文
1. 学校に行きます（学校へ行く - 目的地）
2. 教室で勉強します（教室で勉強する - 場所）
3. 7時に起きます（7時に起きる - 時点）
4. バスで行きます（バスで行く - 方法）

## 練習問題
1. 明日、図書館___本を借ります。
   A) に  B) で  C) を

## 推薦リソース
- [動画] Tae Kim's Grammar Guide - Particles
- [練習] JLPT Sensei - に/で 練習
```

#### 機能 4：学習パス推薦 🎯 **中優先度**

**機能説明**

エージェントがユーザープロファイルを維持し、次に練習すべき内容を自律的に推薦します。

**エージェント自律行動**
- 学習履歴と進捗を追跡
- 弱点の分布を分析
- 最適な練習順序を推薦
- 適切な難易度レベルを評価

**ユーザーストーリー**
```
日本語学習者として
次に何を学ぶべきかを知りたい
計画的に向上するために
```

**受け入れ基準**
- [ ] アプリを開くたびに「今日の推薦」を表示
- [ ] 推薦にはシーン選択と難易度を含む
- [ ] 推薦理由を説明
- [ ] ユーザーが他のコンテンツを自由に選択可能
- [ ] 推薦は少なくとも直近 3 回の練習データに基づく

**推薦カード例**
```
🎯 今日の推薦

シーン：ショッピング
難易度：⭐⭐⭐ (3/5)

推薦理由：
• 前回の「レストラン」シーンのパフォーマンスが良好（82点）
• ただし助詞の使用にはまだ強化が必要
• 「ショッピング」シーンには助詞練習の機会が多い

[練習を開始] [他のシーンを選択]
```

## 5. シーンコンテンツ設計

### 5.1 レストラン注文シーン

**シーン設定**
- 場所：日本料理店
- エージェントの役割：ウェイター（店員）
- ユーザーの役割：お客様

**難易度レベル**

| 難易度 | 会話複雑度 | 語彙範囲 | 文法要件 |
|--------|------------|----------|----------|
| 1 | シンプルな注文 | 基本的な料理名 | です/ます |
| 2 | おすすめを尋ねる | 形容詞、数量詞 | ～てください |
| 3 | 特別なリクエスト | 調味料、アレルギー | ～ないでください |
| 4 | 複雑なやり取り | セット、割引 | 敬語基礎 |
| 5 | 自然な会話 | 慣用表現 | 敬語上級 |

**典型的な会話フロー**

```
難易度 1 の例：

Agent: いらっしゃいませ。何名様ですか。
       (欢迎光临。请问几位？)

User:  一人です。
       (一个人。)

Agent: かしこまりました。こちらへどうぞ。
       ご注文はお決まりですか。
       (好的。这边请。请问要点什么？)

User:  ラーメンをください。
       (请给我拉面。)

Agent: かしこまりました。お飲み物はいかがですか。
       (好的。需要饮料吗？)
```

- エージェントの役割：店員

**典型的なトピック**

### 5.3 自己紹介シーン

**典型的なトピック**

## 9. リスクと対策

### 9.1 技術リスク

| リスク | 影響 | 確率 | 対策 |
|--------|------|------|------|
| Gemini API レート制限 | 高 | 中 | より高いクォータを申請；リクエストキュー；フォールバック方案 |
| 音声認識精度が低い | 中 | 中 | テキスト入力も同時サポート；オーディオ前処理の最適化 |
| Cloud Run コールドスタートが遅い | 中 | 高 | 最小インスタンスを維持；プリウォーミング機構を設定 |

### 9.2 製品リスク

| リスク | 影響 | 確率 | 対策 |
|--------|------|------|------|
| ユーザーが日本語を話せない | 高 | 中 | ヒント機能を提供；ローマ字入力をサポート |
| 会話が不自然 | 中 | 中 | プロンプトを最適化；フィードバックを収集してイテレーション |
| 分析が不正確 | 中 | 低 | サンプルを手動レビュー；継続的にモデルを最適化 |

### 9.3 提出リスク

| リスク | 対策 |
|--------|------|
| デモ動画の品質 | 複数回録画し、最良のバージョンを選択；バックアップ環境を準備 |
| オンラインサービスの安定性 | 審査期間中はサービスを利用可能に維持；監視アラート |
| 機能デモが不完全 | 動画で全コア機能をカバー；オンライン試用入口を提供 |

## 13. 付録

### 13.1 重要決定記録

**決定 1：なぜ他の言語ではなく日本語学習を選んだのか？**
- 日本のローカル市場でのニーズがより強い
- マルチモーダル能力（音声）を発揮する余地が大きい
- 日本語文法の複雑さは AI 分析能力を示すのに適している

**決定 2：なぜ Cloud Functions ではなく Cloud Run を使うのか？**
- 会話状態の維持が必要（ステートフルサービス）
- WebSocket サポート（リアルタイム音声）
- より柔軟なデプロイ方式

**決定 3：なぜプリセットではなく教材を生成するのか？**
- 開発時間の節約
- AI 能力の実証
- 真の個別化
